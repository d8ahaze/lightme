Как это работает?

Интерфейсы

Реализации, стоящие за интерфейсами, это самостоятельные программы, логика которых рассмотрена в соответствующих метриках секции их анализа и сбора. cli: omelet mysite.com. Web api: https://zxcvx.cc/packs/omelet/host?domain=mysite.com

Анализ и сбор факторов ранжирования

Каждый параметр представляет собой столбцы в базе. Поскольку доступ к DNS записям осуществляется владельцами сайта для каждого параметра реализуется своя логика накопления информации о нем. Если же пользователь предоставляет доступ, указывая наш ключ в TXT записи, то возможно использование api популярных поисковиков. Также, в случае если пользователю потребуется сравнительный анализ, относительно других сайтов категории исходного, то потребуется плата от пользователя для доступа к данным траффика других ресурсов. Дальнейший анализ происходит по этим извлеченным и затем загруженным данным.

Специфика отдельных поисковых систем

Собранные нами параметры, хранятся не только в соответствии со строками домен/файл базы данных, но и имеют "условный вес" в поле значения для отделно взятых поисковых систем. Это способствует созданию более точных рекомендаций относительно поисковых систем, которые можно рассматривать в данном случае как особый вид метрики.

Bing features:

  • Instant view: allows preview top web sites from response
  • Semantic analysis: interpreting of content
  • AI chat bot: return complete, ready-to-use answer generated from top web sites.
